# penjelasan singkat terkait file act yang telah di push kedalam repository.
Gambar tersebut memperlihatkan proses penggunaan Git untuk mengelola proyek dan menghubungkannya ke repository GitHub. Diawali dengan inisialisasi repository lokal (git init), kemudian mengatur remote origin ke URL repository GitHub, membuat atau memastikan branch bernama main, lalu menambah dan meng-commit file proyek dengan pesan commit (misalnya “first commit”). Setelah itu, branch main didorong (push) ke repository GitHub sehingga proyek lokal tersimpan di server GitHub, dan pada tahap berikutnya dilakukan git pull untuk mengambil perubahan terbaru dari repository (misalnya penambahan file README.md) agar kondisi repository lokal dan remote tetap sinkron.
